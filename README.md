##  Як влаштований та працює цей проєкт

Проєкт є статичним веб-сайтом групи, де кожен студент має власну інтерактивну картку-візитку. 

### Архітектура проєкту
1. `index.html` та `aboutus.html` — основна розмітка сайту, що містить інформацію про предмети, список тем лекцій та адаптивну сітку студентів.
2. `style.css` — файл стилів, який відповідає за зовнішній вигляд, адаптивність під мобільні екрани та гнучку верстку за допомогою системи CSS Flexbox (`align-items stretch`).
3. Папка `avatars` — сховище локальних зображень студентів, які використовуються як аватарки в профілях.

##  Чого ми навчилися в процесі (Our Learning Journey).

Під час виконання цієї практичної роботи ми освоїли базовий стек інструментів сучасного розробника (Modern Dev Сhain)
 Робота з Git через CLI (Термінал) навчилися клонувати репозиторії, керувати локальними гілками (`git checkout -b`), фіксувати зміни (`git commit`) та безпечно завантажувати їх на сервер за допомогою `force push`.
 Управління проєктами (Project Management) розібралися з методологією Kanban у Trello, навчилися працювати з колонками `Backlog`, `In Progress`, `Ready for Review` та створювати професійні баг-репорти (Bug Reports) із градацією серйозності (Severity).
 Співпраця через GitHub опанували механізм створення Pull Request (PR), перевірки конфліктів та злиття гілок.
 Верстка та Баг-фіксинг розібралися, як працює вирівнювання Flexbox елементів і як виправляти помилки, коли сітка карток відображається нерівно.
